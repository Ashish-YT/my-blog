Quick Concepts

  • Packet — a small unit of data.
  • Bandwidth — capacity of a link.
  • Latency — delay between sender & receiver.
  • Routing — path selection across a network.
Tip: Hover cards to see micro-interactions.

1. What is a Packet?

Packets contain headers and payloads. Visualize them as envelopes traveling between nodes; headers guide delivery, payload carries the content.

2. How Routing Works

Routers learn network topology and choose best paths. Routing protocols exchange small messages to keep the network updated.

3. TCP vs UDP

TCP guarantees delivery (retransmits lost packets). UDP is faster but unreliable — used for streaming where speed matters.

4. Bandwidth & Throughput

Bandwidth is theoretical maximum; throughput is actual data successfully delivered over time, affected by congestion and loss.

5. Latency & Jitter

Latency is round-trip time. Jitter is variation in packet arrival — critical for real-time applications like VoIP.

6. Error Detection

Checksums and CRCs help detect corrupted packets so systems can request retransmission or discard data.
